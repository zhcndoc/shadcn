## 2025年10月 - 新组件

这次的新组件，我关注的是我们每天构建的那些乏味内容，那些我们反复重建的东西，并制作了可以真正复用的抽象组件。

**这些组件兼容所有组件库，Radix、Base UI、React Aria，随便什么库。复制粘贴到你的项目即可使用。**

- [Spinner](#spinner)：加载状态指示器。
- [Kbd](#kbd)：显示键盘按键或按键组合。
- [按钮组 (Button Group)](#button-group)：一组相关按钮，用于操作和拆分按钮。
- [输入组 (Input Group)](#input-group)：带图标、按钮、标签等的输入框组合。
- [Field](#field)：一个组件，覆盖你所有的表单需求。
- [Item](#item)：显示项目列表、卡片等。
- [Empty](#empty)：用于空状态显示。

### 新特性

- [命名空间注册表](#namespaced-registries) - 支持通过 `@registry/name` 格式安装组件。
- [私有注册表](#private-registries) - 通过高级认证保障注册表安全。
- [搜索与发现](#search--discovery) - 新命令帮助你查找和预览代码。
- [MCP 服务器](#mcp-server) - 所有注册表的 MCP 服务器。
- [整体提速](#faster-everything) - 重新构建的注册表解析引擎。
- [改进错误处理](#improved-error-handling) - 提供更友好的错误提示给用户及 LLM。
- [升级指南](#upgrade-guide) - 现有用户迁移说明。

### 命名空间注册表

3.0 最大变化是支持命名空间注册表。你可以从社区注册表、公司私有注册表或内部注册表安装组件，格式是 `@registry/name`。

这让跨团队和跨项目分发代码更容易。

在 `components.json` 配置注册表：

```json title="components.json"
{
  "registries": {
    "@acme": "https://acme.com/r/{name}.json",
    "@internal": {
      "url": "https://registry.company.com/{name}",
      "headers": {
        "Authorization": "Bearer ${REGISTRY_TOKEN}"
      }
    }
  }
}
```

然后用 `@registry/name` 格式安装组件：

```bash
npx shadcn add @acme/button @internal/auth-system
```

完全去中心化，无中央注册机构。你可以自由创建命名空间，按团队需求组织组件。

```json title="components.json" showLineNumbers
{
  "registries": {
    "@design": "https://registry.company.com/design/{name}.json",
    "@engineering": "https://registry.company.com/eng/{name}.json",
    "@marketing": "https://registry.company.com/marketing/{name}.json"
  }
}
```

组件还可以依赖不同注册表的资源，解析及安装全部自动完成。

```json title="registry-item.json" showLineNumbers
{
  "name": "dashboard",
  "type": "registry:block",
  "registryDependencies": [
    "@shadcn/card", // 默认注册表
    "@v0/chart", // v0 注册表
    "@acme/data-table", // acme 注册表
    "@lib/data-fetcher", // 工具库
    "@ai/analytics-prompt" // AI prompt 资源
  ]
}
```

### 私有注册表

需要保密组件？没问题。支持用 token、API key、或自定义请求头来配置认证：

```json title="components.json"
{
  "registries": {
    "@private": {
      "url": "https://registry.company.com/{name}.json",
      "headers": {
        "Authorization": "Bearer ${REGISTRY_TOKEN}"
      }
    }
  }
}
```

私有组件保持私密，特别适合企业团队的专有 UI 库。

我们支持所有主流认证方式：Basic Auth、Bearer Token、API Key 查询参数、定制请求头。

详见 [认证文档](/docs/registry/authentication)。

### 搜索与发现

三条新命令让你轻松找到所需：

1. 预览注册表中的项目再安装

```bash
npx shadcn view @acme/auth-system
```

2. 在注册表中搜索项目

```bash
npx shadcn search @tweakcn -q "dark"
```

3. 列出某注册表的所有项目


预览组件，跨多注册表搜索，提前查看代码和依赖。

### 改进错误处理

注册表开发者现在可以提供自定义错误信息，帮助用户（与 LLM）定位问题。CLI 展示有用的操作建议：

```txt
未知注册表 "@acme"。确保在 components.json 中定义如下：
{
  "registries": {
    "@acme": "[REGISTRY_URL]"
  }
}
```

缺少环境变量时，CLI 会精准提示：

```txt
注册表 "@private" 需要以下环境变量：
  • REGISTRY_TOKEN

请将所需变量添加至 .env 或 .env.local 文件。
```

注册表作者可以在响应中加入自定义错误，帮助用户和 AI 快速理解并修复问题。

```txt
错误：
你无权访问 http://example.com/r/component 中的项。

消息：
[未授权] 你的 API key 已过期。请访问 https://example.com/api/renew-key 更新。
```

### 升级指南

最棒的是：没有破坏性变更。你现有的 `components.json` 和已安装组件照样可用。

开发者若用到程序化 API，部分函数已废弃，推荐用替代方案：

- `fetchRegistry` → `getRegistry`
- `resolveRegistryTree` → `resolveRegistryItems`
- schema 包从 `shadcn/registry` 移至 `shadcn/schema`

```diff
- import { registryItemSchema } from "shadcn/registry"
+ import { registryItemSchema } from "shadcn/schema"
```

就这样，其他一切皆正常工作。

## 2025年6月 - `radix-ui`

我们新增命令支持迁移到新版 `radix-ui` 包。该命令会将所有 `@radix-ui/react-*` 的导入替换为 `radix-ui`。

```bash
npx shadcn@latest migrate radix
```

它会自动更新所有 `ui` 组件里的导入，并安装 `radix-ui` 依赖。

```diff showLineNumbers title="components/ui/alert-dialog.tsx"
- import * as AlertDialogPrimitive from "@radix-ui/react-dialog"
+ import { AlertDialog as AlertDialogPrimitive } from "radix-ui"
```

命令执行后，请测试组件和项目。

**注意：** 若后续新增组件，要再次运行迁移命令更新导入。
