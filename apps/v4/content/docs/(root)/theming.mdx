想要可视化地构建你的主题吗？使用 [shadcn/create](/create) 预览颜色、圆角、字体和图标，然后为你的项目生成预设。

我们使用并推荐使用 CSS 变量进行主题化。

这为你提供了语义化的主题令牌，如 `background`、`foreground` 和 `primary`，组件默认使用这些令牌。在 CSS 中覆盖这些令牌即可更改应用的外观，而无需重写组件类。

```tsx /bg-background/ /text-foreground/
<div className="bg-background text-foreground" />
```

要在主题化中使用 CSS 变量，请在 `components.json` 文件中将 `tailwind.cssVariables` 设置为 `true`。这是默认值。

```json {8} title="components.json" showLineNumbers
{
  "style": "base-nova",
  "rsc": true,
  "tailwind": {
    "config": "",
    "css": "app/globals.css",
    "baseColor": "neutral",
    "cssVariables": true
  }
}
```

Tailwind 将这些令牌映射为实用工具类，如 `bg-background`、`text-foreground`、`border-border` 和 `ring-ring`。

深色模式通过在 `.dark` 选择器内覆盖相同的令牌来工作。请参阅 [深色模式文档](/docs/dark-mode/next) 了解如何添加主题提供者并切换 `.dark` 类。

## 令牌约定

我们使用语义化的背景和前景配对。基础令牌控制表面颜色，而 `-foreground` 令牌控制位于该表面上的文本和图标颜色。

<Callout className="mt-4">

表面令牌省略了背景后缀。例如，`primary` 与 `primary-foreground` 配对。

</Callout>

假设有如下 CSS 变量：

```css
--primary: oklch(0.205 0 0);
--primary-foreground: oklch(0.985 0 0);
```

那么下面组件的背景色将是 `var(--primary)`，文本颜色为 `var(--primary-foreground)`。

```tsx
<div className="bg-primary text-primary-foreground">Hello</div>
```

## 主题令牌

这些令牌位于你的 CSS 文件中的 `:root` 和 `.dark` 下。

| Token                                            | 控制内容                                       | 使用者                                                                      |
| ------------------------------------------------ | ------------------------------------------------------ | ---------------------------------------------------------------------------- |
| `background` / `foreground`                      | 默认应用背景和文本颜色。             | 页面外壳、页面部分和默认文本。                             |
| `card` / `card-foreground`                       | 层级表面及其内部内容。         | `Card`、仪表板面板、设置面板。                                   |
| `popover` / `popover-foreground`                 | 浮动表面及其内部内容。         | `Popover`、`DropdownMenu`、`ContextMenu` 和其他覆盖层。                |
| `primary` / `primary-foreground`                 | 高强调操作和品牌表面。              | 默认 `Button`、选中状态、徽章和活跃强调色。               |
| `secondary` / `secondary-foreground`             | 低强调填充操作和支持表面。 | 次要按钮、次要徽章和支持性 UI。                      |
| `muted` / `muted-foreground`                     | 微妙表面和低强调内容。            | 描述、占位符、空状态、辅助文本和柔和表面。 |
| `accent` / `accent-foreground`                   | 交互 hover、focus 和 active 表面。         | Ghost 按钮、菜单高亮状态、悬停行和选中项。      |
| `destructive`                                    | 破坏性操作和错误强调。                | 破坏性按钮、无效状态和破坏性菜单项。             |
| `border`                                         | 默认边框和分隔线。                        | 卡片、菜单、表格、分隔线和布局分割线。                       |
| `input`                                          | 表单控件边框和输入表面处理。      | `Input`、`Textarea`、`Select` 和轮廓样式控件。                   |
| `ring`                                           | 焦点环和轮廓。                              | 按钮、输入框、复选框、菜单和其他可聚焦控件。            |
| `chart-1` ... `chart-5`                          | 默认图表调色板。                             | 图表和基于图表的仪表板块。                                    |
| `sidebar` / `sidebar-foreground`                 | 基础侧边栏表面和默认侧边栏文本。     | `Sidebar` 容器及其默认内容。                             |
| `sidebar-primary` / `sidebar-primary-foreground` | 侧边栏内的高强调操作。              | 活跃项、图标块、徽章和侧边栏 CTA。                          |
| `sidebar-accent` / `sidebar-accent-foreground`   | 侧边栏内的悬停和选中状态。          | 侧边栏菜单悬停状态、打开项和交互行。                 |
| `sidebar-border`                                 | 侧边栏特定的边框和分隔线。               | 侧边栏头部、组和内部分隔线。                              |
| `sidebar-ring`                                   | 侧边栏特定的焦点环。                          | 侧边栏内聚焦的控件。                                         |
| `radius`                                         | 基础圆角比例。                          | 卡片、输入框、按钮、popovers 和派生的 `radius-*` 令牌。         |

<Callout className="mt-4">

图表令牌在 [图表主题文档](/docs/components/chart#theming) 中有更详细的介绍。

</Callout>

## 圆角比例

`--radius` 是你主题的基础圆角令牌。

我们从中派生出一个小的圆角比例，以便组件可以使用一致的圆角大小，同时共享单一事实来源。

```css title="app/globals.css" showLineNumbers
@theme inline {
  --radius-sm: calc(var(--radius) * 0.6);
  --radius-md: calc(var(--radius) * 0.8);
  --radius-lg: var(--radius);
  --radius-xl: calc(var(--radius) * 1.4);
  --radius-2xl: calc(var(--radius) * 1.8);
  --radius-3xl: calc(var(--radius) * 2.2);
  --radius-4xl: calc(var(--radius) * 2.6);
}
```

这意味着：

- `radius-lg` 是基础值。
- 较小的圆角从 `--radius` 向下缩放。
- 较大的圆角从 `--radius` 向上缩放。
- 更改 `--radius` 会更新整个圆角比例。

## 添加新令牌

要添加新令牌，请在 `:root` 和 `.dark` 下定义它，然后使用 `@theme inline` 将其暴露给 Tailwind。

```css title="app/globals.css" showLineNumbers
:root {
  --warning: oklch(0.84 0.16 84);
  --warning-foreground: oklch(0.28 0.07 46);
}

.dark {
  --warning: oklch(0.41 0.11 46);
  --warning-foreground: oklch(0.99 0.02 95);
}

@theme inline {
  --color-warning: var(--warning);
  --color-warning-foreground: var(--warning-foreground);
}
```

你现在可以在组件中使用 `bg-warning` 和 `text-warning-foreground`。

```tsx /bg-warning/ /text-warning-foreground/
<div className="bg-warning text-warning-foreground" />
```

## 基础颜色

`tailwind.baseColor` 控制当你运行 `init` 或使用预设时为项目生成的默认令牌值。

可用的基础颜色有：**Neutral**、**Stone**、**Zinc**、**Mauve**、**Olive**、**Mist** 和 **Taupe**。

## 不使用 CSS 变量

如果您不想使用 CSS 变量，CLI 可以改为生成使用内联 Tailwind 颜色工具类的组件。

```bash
npx shadcn@latest init --no-css-variables
```

这会在您的 `components.json` 文件中将 `tailwind.cssVariables` 设置为 `false`。

```tsx /bg-zinc-950/ /text-zinc-50/ /dark:bg-white/ /dark:text-zinc-950/
<div className="bg-zinc-950 text-zinc-50 dark:bg-white dark:text-zinc-950" />
```

<Callout className="mt-4">

这是一个安装时的选择。要切换现有项目，请删除并重新安装您的组件。

</Callout>
