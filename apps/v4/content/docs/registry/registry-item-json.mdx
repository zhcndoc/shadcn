`registry-item.json` 模式用于定义您的自定义注册表项。

```json title="registry-item.json" showLineNumbers
{
  "$schema": "https://ui.shadcn.com/schema/registry-item.json",
  "name": "hello-world",
  "type": "registry:block",
  "title": "Hello World",
  "description": "一个简单的 Hello World 组件。",
  "registryDependencies": [
    "button",
    "@acme/input-form",
    "https://example.com/r/foo"
  ],
  "dependencies": ["is-even@3.0.0", "motion"],
  "devDependencies": ["tw-animate-css"],
  "files": [
    {
      "path": "registry/new-york/hello-world/hello-world.tsx",
      "type": "registry:component"
    },
    {
      "path": "registry/new-york/hello-world/use-hello-world.ts",
      "type": "registry:hook"
    }
  ],
  "cssVars": {
    "theme": {
      "font-heading": "Poppins, sans-serif"
    },
    "light": {
      "brand": "oklch(0.205 0.015 18)"
    },
    "dark": {
      "brand": "oklch(0.205 0.015 18)"
    }
  }
}
```

## 定义

您可以在[这里](https://ui.shadcn.com/schema/registry-item.json)查看 `registry-item.json` 的 JSON Schema。

### $schema

`$schema` 属性用于指定 `registry-item.json` 文件的模式。

```json title="registry-item.json" showLineNumbers
{
  "$schema": "https://ui.shadcn.com/schema/registry-item.json"
}
```

### name

项目的名称。用于在注册表中识别该项目。应在您的注册表中唯一。

```json title="registry-item.json" showLineNumbers
{
  "name": "hello-world"
}
```

### title

注册表项目的可读标题。保持简短且具描述性。

```json title="registry-item.json" showLineNumbers
{
  "title": "Hello World"
}
```

### description

注册表项目的描述。可以比 `title` 更长且更详细。

```json title="registry-item.json" showLineNumbers
{
  "description": "一个简单的 Hello World 组件。"
}
```

### type

`type` 属性用于指定注册表项目的类型。用于确定项目的类型以及在项目中的安装位置。

```json title="registry-item.json" showLineNumbers
{
  "type": "registry:block"
}
```

支持的类型如下：

| 类型                  | 描述                                           |
| --------------------- | ---------------------------------------------- |
| `registry:base`       | 用于整个设计系统。                              |
| `registry:block`      | 用于包含多个文件的复杂组件。                    |
| `registry:component`  | 用于简单组件。                                  |
| `registry:font`       | 用于字体。                                      |
| `registry:lib`        | 用于库和工具函数。                              |
| `registry:hook`       | 用于钩子函数。                                  |
| `registry:ui`         | 用于 UI 组件及单文件基础组件。                   |
| `registry:page`       | 用于页面或基于文件的路由。                       |
| `registry:file`       | 用于杂项文件。                                  |
| `registry:style`      | 用于注册表样式，如 `new-york`。                 |
| `registry:theme`      | 用于主题。                                      |
| `registry:item`       | 用于通用注册表项。                              |

### author

`author` 属性用于指定注册表项目的作者。

可以是项目独有，也可以与注册表的作者相同。

```json title="registry-item.json" showLineNumbers
{
  "author": "John Doe <john@doe.com>"
}
```

### dependencies

`dependencies` 属性用于指定注册表项的依赖，针对 `npm` 包。

使用 `@version` 指定版本号。

```json title="registry-item.json" showLineNumbers
{
  "dependencies": [
    "@radix-ui/react-accordion",
    "zod",
    "lucide-react",
    "name@1.0.2"
  ]
}
```

### devDependencies

`devDependencies` 属性用于指定注册表项的开发依赖。这些是仅在开发期间需要的 `npm` 包。

使用 `@version` 指定包的版本号。

```json title="registry-item.json" showLineNumbers
{
  "devDependencies": ["tw-animate-css", "name@1.2.0"]
}
```

### registryDependencies

用于注册表依赖。每一项都是一个条目地址。

- 对于 `shadcn/ui` 注册表项，例如 `button`、`input`、`select` 等，请使用名称，例如 `['button', 'input', 'select']`。
- 对于带命名空间的注册表项，请使用 `@namespace/item-name`，例如 `['@acme/input-form']`。
- 对于 GitHub 注册表项，请使用 `owner/repo/item-name`，例如 `['acme/ui/button']`。对于已发布的注册表，优先使用标签或完整提交 SHA，例如 `['acme/ui/button#v1.2.0']`。
- 对于自定义注册表项，请使用注册表项的 URL，例如 `['https://example.com/r/hello-world.json']`。
- 对于本地注册表项文件，请使用文件路径，例如 `['./hello-world.json']`。

```json title="registry-item.json" showLineNumbers
{
  "registryDependencies": [
    "button",
    "@acme/input-form",
    "acme/ui/button#v1.2.0",
    "https://example.com/r/editor.json",
    "./editor.json"
  ]
}
```

注意：裸名称会保留其原有行为。`button` 表示内置的 shadcn `button` 项，而不是同一个 GitHub 仓库中的项。对于同仓库的 GitHub 依赖，请使用完整的 GitHub 条目地址。

引用不会跨依赖继承。如果某个 GitHub 依赖应当具备可复现性，请将该依赖固定到其自己的标签或完整提交 SHA。

更多信息请参阅 [GitHub registry](/docs/registry/github) 文档。

### files

`files` 属性用于指定注册表项目的文件。每个文件拥有 `path`、`type` 和可选的 `target` 属性。

**`target` 属性是 `registry:page` 和 `registry:file` 类型必须的。**

```json title="registry-item.json" showLineNumbers
{
  "files": [
    {
      "path": "registry/new-york/hello-world/page.tsx",
      "type": "registry:page",
      "target": "app/hello/page.tsx"
    },
    {
      "path": "registry/new-york/hello-world/hello-world.tsx",
      "type": "registry:component"
    },
    {
      "path": "registry/new-york/hello-world/use-hello-world.ts",
      "type": "registry:hook"
    },
    {
      "path": "registry/new-york/hello-world/.env",
      "type": "registry:file",
      "target": "~/.env"
    }
  ]
}
```

#### path

`path` 属性用于指定文件在注册表中的路径。此路径被构建脚本用来解析、转换并生成注册表 JSON 负载。

#### type

`type` 属性用于指定文件的类型。具体请参见[type](#type)节。

#### target

`target` 属性表示文件应放置在项目中的位置。此属性为可选，仅对 `registry:page` 和 `registry:file` 类型必需。

默认情况下，`shadcn` CLI 会读取项目的 `components.json` 文件来确定目标路径。对于某些文件，如路由或配置文件，可以手动指定目标路径。

使用 `~` 表示项目根目录，例如 `~/foo.config.js`。

您还可以使用注册表目标占位符，将文件放置到由用户 `components.json` 配置的目录中。这些占位符仅支持出现在 `target` 的开头，并且与项目的导入前缀无关。例如，无论项目使用 `@/`、`#`、包导入还是工作区导出导入组件，`@ui/button.tsx` 都可以正常工作。

| 占位符         | 解析为               |
| -------------- | -------------------- |
| `@components/` | `aliases.components` |
| `@ui/`         | `aliases.ui`         |
| `@lib/`        | `aliases.lib`        |
| `@hooks/`      | `aliases.hooks`      |

当您希望某个注册表项安装到项目中配置好的 shadcn 目录，而不是硬编码 `components`、`src` 或工作区包路径时，请使用这些占位符。占位符之后的内容会被保留，因此 `@ui/ai/prompt-input.tsx` 会安装到用户配置的 `ui` 目录下的 `ai/prompt-input.tsx`。

```json title="registry-item.json" showLineNumbers
{
  "files": [
    {
      "path": "registry/new-york/example/button.tsx",
      "type": "registry:ui",
      "target": "@ui/button.tsx"
    },
    {
      "path": "registry/new-york/example/prompt-input.tsx",
      "type": "registry:ui",
      "target": "@ui/ai/prompt-input.tsx"
    },
    {
      "path": "registry/new-york/example/card.tsx",
      "type": "registry:component",
      "target": "@components/card.tsx"
    },
    {
      "path": "registry/new-york/example/helper.ts",
      "type": "registry:lib",
      "target": "@lib/helper.ts"
    },
    {
      "path": "registry/new-york/example/use-demo.ts",
      "type": "registry:hook",
      "target": "@hooks/use-demo.ts"
    }
  ]
}
```

`target` 属性决定文件的写入位置。它可以指向与文件 `type` 不同的 shadcn 目录。

```json title="registry-item.json" showLineNumbers
{
  "files": [
    {
      "path": "registry/new-york/example/format-date.ts",
      "type": "registry:ui",
      "target": "@lib/format-date.ts"
    }
  ]
}
```

未知占位符会被视为普通目标路径。例如，`@foo/bar.ts` 会写入为 `foo/bar.ts`。嵌入式占位符，例如 `components/@ui/button.tsx`，也会被视为普通路径。

<Callout>
  `@utils/` 不受支持，因为 `utils` 指向的是文件，而不是目录。
</Callout>

### tailwind

**弃用：** Tailwind V4 项目建议使用 `cssVars.theme` 来代替。

`tailwind` 属性用于 Tailwind 配置，例如 `theme`、`plugins` 和 `content`。

您可以通过 `tailwind.config` 为注册表项添加颜色、动画和插件。

```json title="registry-item.json" showLineNumbers
{
  "tailwind": {
    "config": {
      "theme": {
        "extend": {
          "colors": {
            "brand": "hsl(var(--brand))"
          },
          "keyframes": {
            "wiggle": {
              "0%, 100%": { "transform": "rotate(-3deg)" },
              "50%": { "transform": "rotate(3deg)" }
            }
          },
          "animation": {
            "wiggle": "wiggle 1s ease-in-out infinite"
          }
        }
      }
    }
  }
}
```

### cssVars

用于定义注册表项目的 CSS 变量。

```json title="registry-item.json" showLineNumbers
{
  "cssVars": {
    "theme": {
      "font-heading": "Poppins, sans-serif"
    },
    "light": {
      "brand": "20 14.3% 4.1%",
      "radius": "0.5rem"
    },
    "dark": {
      "brand": "20 14.3% 4.1%"
    }
  }
}
```

### css

使用 `css` 向项目的 CSS 文件添加新规则，如 `@layer base`、`@layer components`、`@utility`、`@keyframes`、`@plugin` 等。

```json title="registry-item.json" showLineNumbers
{
  "css": {
    "@plugin @tailwindcss/typography": {},
    "@plugin foo": {},
    "@layer base": {
      "body": {
        "font-size": "var(--text-base)",
        "line-height": "1.5"
      }
    },
    "@layer components": {
      "button": {
        "background-color": "var(--color-primary)",
        "color": "var(--color-white)"
      }
    },
    "@utility text-magic": {
      "font-size": "var(--text-base)",
      "line-height": "1.5"
    },
    "@keyframes wiggle": {
      "0%, 100%": {
        "transform": "rotate(-3deg)"
      },
      "50%": {
        "transform": "rotate(3deg)"
      }
    }
  }
}
```

### envVars

使用 `envVars` 向注册表项目添加环境变量。

```json title="registry-item.json" showLineNumbers
{
  "envVars": {
    "NEXT_PUBLIC_APP_URL": "http://localhost:4000",
    "DATABASE_URL": "postgresql://postgres:postgres@localhost:5432/postgres",
    "OPENAI_API_KEY": ""
  }
}
```

环境变量将被添加到 `.env.local` 或 `.env` 文件。已存在的变量不会被覆盖。

<Callout>

**重要：** 使用 `envVars` 添加开发或示例环境变量。请勿用来添加生产环境变量。

</Callout>

### font

`font` 属性是 `registry:font` 类型项目必需的。它配置字体系列、提供方、导入名称、CSS 变量以及为非 Next.js 项目安装的 npm 包。

```json title="registry-item.json" showLineNumbers
{
  "font": {
    "family": "'Inter Variable', sans-serif",
    "provider": "google",
    "import": "Inter",
    "variable": "--font-sans",
    "subsets": ["latin"],
    "dependency": "@fontsource-variable/inter"
  }
}
```

| 属性         | 类型       | 必需 | 描述                                                                               |
| ------------ | ---------- | ---- | ---------------------------------------------------------------------------------- |
| `family`     | `string`   | 是   | CSS `font-family` 值。                                                             |
| `provider`   | `string`   | 是   | 字体提供方。目前仅支持 `google`。                                                  |
| `import`     | `string`   | 是   | 从 `next/font/google` 导入字体的名称。                                              |
| `variable`   | `string`   | 是   | 字体的 CSS 变量名（例如 `--font-sans`、`--font-mono`）。                           |
| `weight`     | `string[]` | 否   | 要包含的字体粗细数组。                                                             |
| `subsets`    | `string[]` | 否   | 要包含的字体子集数组。                                                             |
| `selector`   | `string`   | 否   | 应用字体的 CSS 选择器。默认为 `html`。                                              |
| `dependency` | `string`   | 否   | 为非 Next.js 项目安装的 npm 包（例如 `@fontsource-variable/inter`）。             |

### docs

使用 `docs` 在通过 CLI 安装注册表项时显示自定义文档或信息。

```json title="registry-item.json" showLineNumbers
{
  "docs": "如需获取 OPENAI_API_KEY，请前往 https://platform.openai.com 注册账户。"
}
```

### categories

使用 `categories` 来组织您的注册表项目。

```json title="registry-item.json" showLineNumbers
{
  "categories": ["sidebar", "dashboard"]
}
```

### meta

使用 `meta` 为您的注册表项添加额外的元数据。您可以添加任何希望在注册表项中可用的键值对。

```json title="registry-item.json" showLineNumbers
{
  "meta": { "foo": "bar" }
}
```
