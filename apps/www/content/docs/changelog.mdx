## 2024 年 8 月 - npx shadcn init

新的 CLI 现已可用。这是一次完整的重写，包含许多新特性和改进。现在您可以使用 `npx shadcn add` 安装组件、主题、钩子、工具等。

这是向您和您的 LLM 可以访问和使用的代码分发迈出的重要一步。

1. 首先，CLI 现在支持所有主要的 React 框架，包括 Next.js、Remix、Vite 和 Laravel。当您初始化新应用时，我们会更新您现有的 Tailwind 文件，而不是覆盖它们。
2. 组件现在可以随附自己的依赖。例如，手风琴组件可以定义其 Tailwind 关键帧。当您将其添加到项目时，我们将相应地更新您的 tailwind.config.ts 文件。
3. 您还可以使用 URL 安装远程组件。`npx shadcn add https://acme.com/registry/navbar.json`。
4. 我们还改进了初始化命令。它支持框架检测，甚至可以用一条命令初始化一个全新的 Next.js 应用。`npx shadcn init`。
5. 我们创建了一个新的模式，您可以用来分发自己的组件注册表。由于它支持 URL，您甚至可以用来分发私有组件。
6. 还有一些额外的更新，比如更好的错误处理和 monorepo 支持。

今天您可以尝试新的 CLI。

```bash
npx shadcn init sidebar-01 login-01
```

### 更新你的项目

要更新现有项目以使用新的 CLI，请更新您的 `components.json` 文件，以包含您 **组件**、**工具**、**UI**、**库** 和 **钩子** 的导入别名。

```json showLineNumbers {7-13} title="components.json"
{
  "$schema": "https://ui.shadcn.com/schema.json",
  "style": "new-york",
  "tailwind": {
    // ...
  },
  "aliases": {
    "components": "@/components",
    "utils": "@/lib/utils",
    "ui": "@/components/ui",
    "lib": "@/lib",
    "hooks": "@/hooks"
  }
}
```

如果您使用不同的导入别名前缀，例如 `~`，请将 `@` 替换为您的前缀。

```bash

```

当您运行 `init` 命令时，将会询问您一些问题以配置 `components.json`：

```txt showLineNumbers
您希望使用什么样式？ › 默认
您希望使用什么颜色作为基础颜色？ › Slate
您的全局 CSS 文件在哪里？ › › app/globals.css

  "aliases": {
    "utils": "~/lib/utils",
    "components": "~/components"
  }
}
```

这意味着您现在可以使用 CLI 进行任何目录结构的操作，包括 `src` 和 `app` 目录。

### 主题使用 CSS 变量或 Tailwind 颜色

您可以选择使用 CSS 变量或 Tailwind CSS 工具类进行主题设置。

当您添加新组件时，CLI 将根据您的 `components.json` 配置自动使用正确的主题设置方法。

#### 工具类

```tsx /bg-zinc-950/ /text-zinc-50/ /dark:bg-white/ /dark:text-zinc-950/
<div className="bg-zinc-950 dark:bg-white" />
```

要使用工具类进行主题设置，请在 `components.json` 文件中将 `tailwind.cssVariables` 设置为 `false`。

```json {6} title="components.json"
{
  "tailwind": {
    "config": "tailwind.config.js",
    "css": "app/globals.css",
    "baseColor": "slate",
    "cssVariables": false
  }
}
```

#### CSS 变量

```tsx /bg-background/ /text-foreground/
<div className="bg-background text-foreground" />
```

要使用 CSS 变量类进行主题设置，请在 `components.json` 文件中将 `tailwind.cssVariables` 设置为 `true`。

```json {6} title="components.json"
{
  "tailwind": {
    "config": "tailwind.config.js",
    "css": "app/globals.css",
    "baseColor": "slate",
    "cssVariables": true
  }
}
```

### 基础颜色

您现在可以配置项目的基础颜色。这将用于生成组件的默认色板。

```json {5} title="components.json"
{
  "tailwind": {
    "config": "tailwind.config.js",
    "css": "app/globals.css",
    "baseColor": "zinc",
    "cssVariables": false
  }
}
```

可以选择 `gray`、`neutral`、`slate`、`stone` 或 `zinc`。

如果您将 `cssVariables` 设置为 `true`，我们将在您的 `globals.css` 文件中将基础颜色设置为 CSS 变量。如果您将 `cssVariables` 设置为 `false`，我们将在组件中内联 Tailwind CSS 工具类。
